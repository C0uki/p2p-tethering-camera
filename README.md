# 📷 P2P テザリングカメラ

スマホのカメラ映像を、同じWi-Fi（テザリング）内のパソコンにリアルタイム送信するWebアプリです。  
インストール不要・ブラウザだけで動作します。

---

## ✨ 特徴

- **ギガを消費しない** — スマホのテザリングに繋いで使えば、モバイル通信を使わずに映像を転送できます
- **FHD対応** — 最大1920×1080 / 30fps でのストリーミングを要求します（端末性能による）
- **背面カメラ自動選択** — スマホのアウトカメラを優先して使用します
- **インストール不要** — HTMLファイルをブラウザで開くだけで動作します
- **P2P通信** — [PeerJS](https://peerjs.com/) を使用したブラウザ間直接通信

---

## 🚀 使い方

### 1. ファイルを開く
`index.html` をパソコンとスマホ、それぞれのブラウザで開きます。

> **ローカルファイルで開く場合の注意**  
> `file://` プロトコルではカメラAPIが動作しないブラウザがあります。  
> その場合は [GitHub Pages](#github-pages-でホストする) や簡易サーバーを使ってください。

### 2. パソコン側（受信者）
1. 「**1. 受信者になる**」をクリック
2. 画面に表示された **ID** をスマホに伝える（コピーボタンで簡単にコピーできます）

### 3. スマホ側（送信者）
1. 「**2. 送信者になる**」をタップ
2. カメラの使用を許可する
3. パソコンに表示された **ID** を入力
4. 「**映像を送信スタート**」をタップ

### 4. 確認
パソコン画面にスマホのカメラ映像が表示されれば成功です 🎉

## 📡 GitHub Pages でホストする

リポジトリを GitHub Pages で公開すると、`https://` 経由でアクセスでき、カメラAPIが安定して動作します。

1. このリポジトリの **Settings** → **Pages** を開く
2. Source を `main` ブランチの `/ (root)` に設定して **Save**
3. 数分後に `https://C0uki.github.io/p2p-tethering-camera/` でアクセス可能になります

## 🛠 技術構成

| 項目 | 内容 |
|---|---|
| 通信方式 | WebRTC（PeerJS経由） |
| シグナリング | PeerJS Cloud Server（デフォルト） |
| NAT越え | STUN（Google STUNサーバー） |
| 依存ライブラリ | [peerjs@1.5.2](https://unpkg.com/peerjs@1.5.2/dist/peerjs.min.js) |
| 動作環境 | Chrome / Safari / Edge（最新版推奨） |

---

## ⚠️ 制限事項

- **TURNサーバー未対応** — 厳しいNAT環境やファイアウォール下では接続できない場合があります
- **PeerJS Cloudサーバー依存** — シグナリングにPeerJSの無料クラウドサーバーを使用しています。本番用途にはセルフホストを推奨します
- **同一ネットワーク推奨** — テザリングホットスポット内（スマホ→パソコン）での使用を想定しています
